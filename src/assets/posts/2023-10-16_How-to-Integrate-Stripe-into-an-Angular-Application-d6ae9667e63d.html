How to Integrate Stripe into an Angular Application

Integrating Stripe into an Angular application involves several steps, from setting up the Stripe account to configuring both the client-side Angular application and the server-side Node.js application. Here‚Äôs a detailed walkthrough with additional information:

Stripe Account

To get started, you need to create a Stripe account. This account allows you to manage your payments, view transactions, and access developer tools. Once you‚Äôve registered your Stripe account, navigate to the developer mode. Here, you‚Äôll find your test publishable and secret keys. These keys are important for secure communication between your Angular application and Stripe.

Angular Application

To work with Stripe in your Angular app, you should install the ngx-stripe npm package. It's a wrapper for Stripe Elements that simplifies the integration process. Here is the full documentation.

npm install ngx-stripe @stripe/stripe-js

In your app.module.ts, import and configure the NgxStripeModule to provide access to Stripe Elements.

import {NgxStripeModule} from "ngx-stripe";

@NgModule({
  declarations: [
    AppComponent
  ],
  imports: [
    BrowserModule,
    AppRoutingModule,
    NgxStripeModule.forRoot()
  ],
  bootstrap: [AppComponent]
})
export class AppModule { }

// stripe-payment.component.ts

import {Component, OnDestroy, OnInit} from '@angular/core';
import { CommonModule } from '@angular/common';

import {NgxStripeModule, StripeCardComponent, StripeFactoryService, StripeInstance, StripeService,} from "ngx-stripe";
import {Subscription, switchMap} from "rxjs";

import {MatButtonModule} from "@angular/material/button";
import {MatInputModule} from "@angular/material/input";
import {HttpClient, HttpResponse} from "@angular/common/http";
import {FormsModule} from "@angular/forms";

@Component({
  selector: 'app-stripe-payment',
  standalone: true,
  imports: [CommonModule, StripeCardComponent, NgxStripeModule, MatInputModule, MatButtonModule, FormsModule],
  templateUrl: './stripe-payment.component.html',
  styleUrls: ['./stripe-payment.component.scss']
})
export class StripePaymentComponent implements OnInit, OnDestroy{

  public stripe!: StripeInstance;
  public stripeAmount!: number;
  public stripePublicKey = 'key';

  private subscriptions: Subscription;

  constructor(
      private http: HttpClient,
      private stripeFactory: StripeFactoryService,
  ) {
    this.subscriptions = new Subscription();
  }

  ngOnDestroy(): void {
    this.subscriptions.unsubscribe();
  }

  ngOnInit(): void {
    // dynamically load the stripe script
    this.stripe = this.stripeFactory.create(this.stripePublicKey);
    // stripe amount is in cents, you should multiply your amount by 100
    this.stripeAmount = 100;
  }

  checkout() {
    const host = 'http://localhost:3000';
    const checkout: Subscription = this.http.post(host + '/create-checkout-session', {data: {amount: this.stripeAmount * 100}}, {observe: 'response'})
        .pipe(
            switchMap((response: HttpResponse<Object>) => {
              const session: IStripeSession = response.body as IStripeSession;
              return this.stripe.redirectToCheckout({ sessionId: session.id });
            })
        )
        .subscribe(result => {
          // If `redirectToCheckout` fails due to a browser or network
          if (result.error) {
            console.log(result.error)
          }
        });
    this.subscriptions.add(checkout);
  }
}

interface IStripeSession {
  id: string;
}

This component integrates with the Stripe payment gateway, allowing users to make secure and convenient payments using their credit cards.

  1. On initialization, the component dynamically loads the Stripe script and sets up a Stripe instance. It also defines the payment amount, which is specified in cents (and should be multiplied by 100 for proper conversion).
  2. The checkout() method is triggered when the user initiates a payment. It sends a request to the server to create a checkout session with the specified amount. The server responds with a session ID, and the component uses this ID to redirect the user to the Stripe checkout page, where they can complete the payment securely.
  3. The component imports various Angular and third-party modules and services, including Angular‚Äôs HttpClient, Stripe-related modules from ‚Äúngx-stripe,‚Äù and Angular Material components for input fields and buttons. This ensures that the necessary dependencies are available for Stripe payment processing.
  4. The stripePublicKey variable is set to the Stripe public key. This key is essential for securely communicating with the Stripe API. You can find this key in your Stripe account.

Node.js Server Application

Firstly, in your Node.js project, you need to install the stripe npm package. The Stripe Node library provides convenient access to the Stripe API from applications written in server-side JavaScript.

npm install stripe

Next, you need to create the Stripe API Server sets up an Express.js application to handle payment transactions through the Stripe payment gateway.

Here is the detailed description of the code above:

  1. The server initializes an Express.js application, creating a robust foundation for handling HTTP requests and managing routes.
  2. The server integrates with the Stripe payment gateway using the Stripe library. This enables users to make payments securely using credit cards.
  3. To handle Cross-Origin Resource Sharing (CORS), the server is configured to allow requests from a specific origin (http://localhost:4200). This is crucial for enabling secure communication with the frontend of a web application.
  4. The server defines a POST endpoint (/create-checkout-session) responsible for creating a Stripe checkout session. This endpoint expects a JSON payload containing payment-related data, such as the payment amount.
  5. Upon receiving a request to create a checkout session, the server utilizes the Stripe library to create a session with relevant details, such as accepted payment methods, line items, and success/cancel URLs.
  6. Upon successful creation of a Stripe checkout session, the server responds with the session details, allowing the client application to proceed with the payment process.
  7. The server securely configures the Stripe secret key (stripeSecretKey), a critical component for establishing a secure and authenticated connection with the Stripe API. You can find this key in your Stripe account.

Testing Your Application

To see your payment system in action, launch both the Angular application and the Node.js server. Make sure they are both running.

When using your Angular application, users will be able to make payments securely through the Stripe checkout.
