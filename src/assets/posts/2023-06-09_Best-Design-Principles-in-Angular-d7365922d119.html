Hey there! So, you want to know how to make the best code design for your Angular application? In this article, we take a closer look at the Interface Segregation Principle (ISP). Let‚Äôs dive into it and see how it can make our code better.

So, ISP is all about not forcing clients to depend on stuff they don‚Äôt need. Imagine you have a bunch of methods in an interface, but different clients only need a few of them. With ISP, we split that big interface into smaller ones, each serving a specific purpose. This way, clients can just depend on the interfaces they actually need, making things more flexible and avoiding unnecessary dependencies.

Example 1: Without ISP in Angular

Imagine we have an EmployeeService interface with methods for adding, updating, deleting, and getting employees. We also have a class called EmployeeManager that implements this interface. The problem is that some clients might only need to get or delete employees, but they still have to depend on the entire EmployeeService interface. Not cool, right? This violates ISP because clients are forced to rely on methods they don't even use.

But fear not, ISP comes to the rescue! In the ‚Äúwith ISP‚Äù scenario, we split the EmployeeService interface into two smaller interfaces: EmployeeReader and EmployeeWriter. The EmployeeReader interface has methods for getting employee info, while the EmployeeWriter interface handles adding, updating, and deleting employees. Now clients can choose to depend only on the interface that suits their needs. Isn't that neat?

  1. Single Responsibility Principle (SRP), says a class should have only one reason to change. It‚Äôs like having a class that focuses on doing just one thing. ISP supports SRP by encouraging smaller interfaces with specific responsibilities. So, each interface follows the SRP and changes to one interface won‚Äôt mess up the others.

  1. Open/Closed Principle (OCP) suggests that software entities should be open for extension but closed for modification. ISP goes hand in hand with OCP by allowing easy extension through new interfaces and implementations without modifying existing code. You can add new clients by just implementing the required interface. No need to change the existing code. Pretty convenient, right?
